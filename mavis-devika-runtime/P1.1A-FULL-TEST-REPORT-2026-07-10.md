## 2. 真实暴露的问题

### 问题 1: LLM 严重不稳定 ⭐⭐⭐⭐⭐

**现象**: qwen3:32b 经常 hang 或超时 (240s 不够)
**频率**: 5 次跑里 4 次撞 timeout 墙
**根本原因**: Ollama 5 分钟自动卸载 + LLM 思考时间长 + 网络波动

**修复方案** (待做):
- 加 LLM 调用 retry 机制 (subprocess level)
- 加 fallback 到 llama3:8b (更快但可能不准)
- 缩短超时但更激进 retry
- 或者用更快的 LLM (Qwen2.5-Coder 之类)

### 问题 2: 不存在文件没合理错误处理 ⭐⭐⭐

**现象**: target_file 指向不存在的文件, runtime 卡在某个早期步骤
**根本原因**: node_05_runner 检测到 target_path 不存在时返回 exit_code=-1,但 Patcher 不会处理这种结构性错误

**修复方案** (待做):
- Coder 节点检测 target_file 不存在 → 立即返回错误,不调 LLM
- Workflow 级别加 early-exit 条件边

### 问题 3: 不给 target_file 时跑完整流程 ⭐⭐

**现象**: 没 target_file 时, 仍然跑完 Planner → Researcher → Coder → Runner → Patcher → Reporter
**根本原因**: Coder 节点没 early-exit, 直接走完

**修复方案** (待做):
- Coder 节点开头检查 target_file, 没就立即返回 "no-op"

### 问题 4: 测试 timeout 配置不合理 ⭐

**现象**: 测试脚本用 240s timeout, 但 runtime 真实跑完需要 4-5 分钟
**修复方案** (已改):
- 把测试 timeout 调到 480s (8 分钟)
