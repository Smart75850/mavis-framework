### 暴露的 2 大 bug (永久 invariant #45)

#### Bug 1: 70% 长度检查对"加一行"小改动太严格

```python
# P3.2 长度检查
if len(full_code) < original_length * 0.7:
    retry  # ❌ 拒绝
```

**问题**: "加一行"任务只需 200 字符, 但原 README 3484 字符, 70% 阈值 = 2438 字符, 200 < 2438 失败。

**修复方向**: 区分"重写文件"和"小改动":
- 重写: 70% 长度检查
- 小改动 (有 "加一行" / "加 docstring" / "加 type hints" 等关键词): 允许变小, 设下限 30%

#### Bug 2: Patcher 14B 中文标点引入 syntax error

Patcher 用 14B 修文件, 但 14B 倾向用中文标点 (句号 '。' / 逗号 '，') 当代码, 引入 syntax error。

**修复方向**:
- Patcher 修后必须再跑 Linter, 失败就回滚 (而不是覆盖)
- 限制 Patcher 14B 不允许输出中文标点到代码块
